## React Flow → Pure SVG Research Brief

### Why React Flow is wrong

| Problem | Root Cause | Impact |
|---------|-----------|--------|
| Arrows point into text | markerEnd orients to Handle position, not path end | Visual jank |
| Labels overlap everything | No layout-aware placement; React Flow's EdgeLabelRenderer is a CSS overlay disconnected from SVG | Cluttered graph |
| Node-edge misalignment | Single centered Handle vs ELK's multi-port routing | Edges connect to wrong position |
| Cannot use ELK ports | Would need dynamic Handle creation per edge, complex refactor | Limits layout quality |
| Dragging disabled | ELK pre-computed paths break on node move | Wasted framework capability |
| 2.4MB bundle | React Flow + its dependencies | Performance |

### What Pure SVG + ELK would look like

```
ELK layout → { nodes: [{x,y,w,h}], edges: [{sections: [{startPoint, bendPoints, endPoint}]}] }
                ↓
SVG render:
  <svg viewBox={computed}>
    <g transform={panZoom}>
      {edges.map(e => <path d={elkPath} stroke={phaseColor} />)}
      {nodes.map(n => <g transform={translate}>
        <circle r={11} fill={color} />
        <text x={28}>{label}</text>
        <text x={28} y={14} class="freq">{frequency}</text>
      </g>)}
      {labels on straight segments, collision-avoided}
    </g>
  </svg>
```

Pan/zoom: single `viewBox` transform or `<g transform>` with wheel/drag handlers.
Selection: SVG click handlers → Zustand store → re-render selected element with highlight.
Minimap: second `<svg>` with scaled-down viewBox.

### Libraries to evaluate

- **Pure SVG** — zero deps, full control, ~200 lines for the renderer
- **D3.js zoom/drag** — `d3-zoom` for pan/zoom, `d3-selection` for DOM manipulation
- **Cytoscape.js** — canvas-based graph viz with ELK extension (`cytoscape-elk`), built-in pan/zoom/selection
- **Sigma.js** — WebGL graph rendering, very fast for large graphs

### Research tasks

1. Read ELK's JSON output format: what exactly do we get for nodes, edges, ports, labels?
2. Prototype a pure SVG renderer (~100 lines) that takes ELK output and renders the graph
3. Implement pan/zoom with wheel + drag (or use d3-zoom)
4. Compare bundle size, render performance, and visual quality vs React Flow
5. Evaluate whether Cytoscape.js's built-in features (selection, hover, minimap) are worth the dependency
